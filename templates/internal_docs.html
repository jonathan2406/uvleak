Documentacion interna — Actualizacion de perfil

Endpoint: PUT /api/profile/update. Acepta campos extendidos (ej. role). Validar con sesion de estudiante.

Sustituye TU_SESSION_TOKEN por el valor de la cookie session_token (DevTools → Application → Cookies). Base URL: {{ base_url }}

Linux / macOS

curl -X PUT '{{ base_url }}api/profile/update' \
  -H 'Content-Type: application/json' \
  -H 'Cookie: session_token=TU_SESSION_TOKEN' \
  -d '{"role":"coordinator"}'

Windows (PowerShell)

curl.exe -X PUT "{{ base_url }}api/profile/update" -H "Content-Type: application/json" -H "Cookie: session_token=TU_SESSION_TOKEN" -d '{"role":"coordinator"}'

Windows (CMD)

curl -X PUT "{{ base_url }}api/profile/update" -H "Content-Type: application/json" -H "Cookie: session_token=TU_SESSION_TOKEN" -d "{^"role^":^"coordinator^"}"

Para este flujo usa coordinator. El rol admin no se puede asignar desde este endpoint.
